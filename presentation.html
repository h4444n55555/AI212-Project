1. System Architecture

The system follows a three-tier design: a React frontend captures images and renders detections, an OR-Tools router dispatches requests to the least-loaded worker using integer programming, and a pool of FastAPI workers each run a singleton YOLOv8m model for GPU inference.

Figure 1. Three-tier system architecture with OR-Tools load-balanced worker pool.

2. Detection Pipeline

Each detection request flows through five stages:

1
React Frontend
Frame capture and upload
Browser encodes webcam/upload as JPEG, sends multipart/form-data POST.
2
OR-Tools Router · :8000
ILP-based load balancing
Solves a min-bottleneck ILP to route the request to the worker with the smallest backlog.
3
FastAPI Worker · asyncio
Non-blocking inference dispatch
Worker offloads inference via asyncio.to_thread(), keeping the event loop free.
4
YOLOv8m · CUDA · FP16
GPU inference
Singleton model runs in half-precision on RTX hardware. Returns bounding boxes and labels.
5
Canvas Renderer
Results rendered on dashboard
Bounding boxes drawn over the video feed. Latency and stats update in real time.

4. REST API

POST /api/detect

Upload an image, receive detections with bounding boxes and latency.

{
  "latency_ms": 38.2,
  "detections": [{
    "label": "person",
    "confidence": 0.92,
    "bbox": [120, 45, 188, 410]
  }]
}
GET /api/health

Live CPU and memory metrics. Polled every 5s by the analytics panel.

{
  "cpu_percent": 24.5,
  "mem_used_gb": 3.78,
  "mem_total_gb": 15.86,
  "status": "ok"
}

5. Installation

1
Install backend dependencies
$cd backend && pip install -r requirements.txt
2
Start FastAPI backend
$python -m uvicorn main:app --port 8000 --reload
3
Install frontend and start Vite
$cd frontend && npm install && npm run dev
4
Open the dashboard
→http://localhost:5173
1
Build and start all services
$docker compose up --build
2
Open the dashboard
→http://localhost:5173
3
Stop all services
$docker compose down
1
Start Minikube
$minikube start
2
Build images inside Minikube
$minikube docker-env | Invoke-Expression
$docker build -t ai212project-backend:latest ./backend
$docker build -t ai212project-frontend:latest ./frontend
3
Apply manifests
$kubectl apply -f k8s/
$kubectl wait --for=condition=ready pod --all --timeout=300s
4
Open the frontend
$minikube service frontend
